# 深圳购房助手小程序优化计划

## 1. 项目概述

### 1.1 核心痛点
用户有购房意向，但直接询问豆包、千问、DeepSeek、Kimi 等 AI 工具无法获得很好的推荐和专业回复。需要一个专门针对深圳购房场景的 AI 推荐工具。

### 1.2 优化目标
- 简化用户操作流程，降低使用门槛
- 强化 AI 智能推荐的核心价值
- 提供专业的购房分析和对比功能
- 零后端成本，纯前端实现

## 2. 当前状态分析

### 2.1 现有功能
- 首页：市场概况、热门小区、功能导航
- 需求页：预算/面积滑块、户型/诉求选择
- 推荐页：AI推荐结果展示
- 详情页：小区详细信息
- 对比页：小区对比功能
- 我的页：收藏、历史记录

### 2.2 现有问题
- 功能过多，核心流程不突出
- 页面跳转复杂，用户体验不流畅
- AI推荐流程不够直观
- 数据量小（仅20个小区），推荐价值有限

### 3.2 页面结构简化

**保留页面（5个）：**
1. **首页** - 极简入口，欢迎+开始按钮+热门小区
2. **AI对话页** - 核心页面，卡片式逐步收集需求
3. **推荐结果页** - 展示AI分析结果和TOP3对比
4. **小区详情页** - 单个小区详细信息
5. **我的** - 收藏列表、历史记录

**移除页面：**
- 需求填写页（被AI对话页替代）
- 对比页（功能合并到推荐结果页）

### 3.3 AI对话页设计

**交互方式：**
- 卡片式布局，每次只显示一个问题卡片
- 用户回答后，卡片收起，显示下一个问题
- 顶部显示进度条（如：问题 3/7）
- 支持返回修改之前的答案

**问题顺序：**
1. **预算范围** - 滑块选择（300-1500万）
2. **面积需求** - 滑块选择（40-200㎡）
3. **户型偏好** - 多选按钮（1室/2室/3室/4室+）
4. **目标区域** - 多选（福田/南山/罗湖等）
5. **工作地点** - 输入框（用于通勤分析）
6. **核心诉求** - 多选（学区/地铁/商业/医疗/环境/升值）
7. **购房目的** - 单选（自住/投资/两者兼顾）

**提交后：**
- 显示"AI正在分析..."加载页
- 调用通义千问API进行智能推荐
- 展示推荐结果

### 3.5 数据方案

**数据量：** 3500个小区（完整数据）
**加载方式：** 首次启动时下载（约1MB）
**存储位置：** 下载后缓存到本地存储
**更新机制：** 每周更新，启动时检查版本

**数据文件：**
- `communities.json` - 小区基础信息
- `prices.json` - 房价数据
- `facilities.json` - 配套数据
- `meta.json` - 元数据（版本号等）

### 3.6 AI Prompt优化

**系统提示词：**

### 4.2 首次启动数据下载

```javascript
// app.js
App({
  onLaunch() {
    this.checkAndDownloadData();
  },
  
  async checkAndDownloadData() {
    const localVersion = wx.getStorageSync('data_version');
    const remoteVersion = await this.getRemoteVersion();
    
    if (localVersion !== remoteVersion) {
      // 显示下载进度
      wx.showLoading({ title: '下载数据中...' });
      
      // 下载数据文件
      await this.downloadDataFile('communities.json');
      await this.downloadDataFile('prices.json');
      await this.downloadDataFile('facilities.json');
      
      wx.setStorageSync('data_version', remoteVersion);
      wx.hideLoading();
    }
  }
});
```
