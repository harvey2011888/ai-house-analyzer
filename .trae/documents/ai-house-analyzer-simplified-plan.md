# 深圳购房助手小程序 - 简化优化计划

## 一、需求分析

### 用户痛点
很多有购房意向的用户想通过AI来选房，但直接去问豆包、千问、deepseek、kimi等都无法得到很好的推荐和专业的回复。

### 解决方案
打造一个极简的微信小程序，核心功能只有一个：**通过AI智能推荐最适合的深圳小区**。

### 优化原则
- **极简**：只保留核心功能，去掉一切非必要的页面和代码
- **专注**：用户打开小程序 → 填写需求 → AI推荐 → 查看详情，三步完成
- **专业**：AI基于深圳真实房价数据给出专业推荐，这是通用AI做不到的

---

## 二、当前状态分析

### 现有问题
1. 页面过多（首页、chat、recommendation、community-detail、profile、compare），功能分散
2. 代码存在潜在问题（data-loader路径、Vant组件引用等）
3. 用户需要多次点击才能看到推荐结果
4. 对比功能使用频率低，增加代码复杂度

### 保留的核心功能
1. **首页** - 欢迎页 + 快速开始按钮 + 热门小区预览
2. **AI需求填写页** - 卡片式逐步收集需求（预算、面积、户型、区域、核心诉求）
3. **推荐结果页** - AI推荐TOP3小区 + 推荐理由
4. **小区详情页** - 基本信息、价格、配套、地图

### 删除的功能
1. ~~profile页面~~ - 去掉收藏、浏览历史、对比功能
2. ~~compare页面~~ - 去掉对比功能
3. ~~storage.js中的收藏/历史/对比逻辑~~ - 简化存储模块
4. ~~底部tabBar~~ - 改为单页导航，去掉"我的"tab

## 五、验证步骤

1. 在微信开发者工具中编译运行
2. 检查首页是否正常显示
3. 点击"开始咨询"进入需求填写页
4. 逐步填写5个问题
5. 点击"开始AI推荐"进入推荐结果页
6. 检查AI推荐结果是否正确显示
7. 点击"查看详情"进入小区详情页
8. 检查小区信息是否正确显示
9. 点击"查看地图位置"测试地图功能
10. 点击"重新咨询"返回需求填写页

## 七、技术要点

1. **千问API调用**：使用 `https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation`
2. **地图功能**：使用微信小程序内置 `wx.openLocation`，免费
3. **数据加载**：通过 `data-loader.js` 内联数据，避免require路径问题
4. **样式统一**：使用CSS变量定义主题色，保持视觉一致
