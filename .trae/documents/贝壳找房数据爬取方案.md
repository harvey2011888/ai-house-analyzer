# 贝壳找房数据爬取方案

## 项目概述

实现从贝壳找房爬取深圳小区数据（基础信息+历史成交记录），使用Node.js + Puppeteer技术栈，定期更新数据以保证AI推荐时候选小区数据的准确性。

## 当前状态分析

### 现有数据体系
- **本地模拟数据**: `data/communities-data.js` - 硬编码的小区基础信息
- **价格数据**: `data/prices-data.js` - 模拟的价格信息
- **政府API数据**: `scripts/download-data.js` - 仅获取区域成交统计，无具体小区
- **AI Prompt数据**: `data/ai-prompts.json` - 手动编写的示例候选数据（不准确）

### 核心问题
1. AI推荐使用的候选小区数据是模拟/示例数据，与真实市场价格偏差大
2. 缺乏自动化数据更新机制
3. 用户看到的房价信息可能不准确

## 目标数据源

**贝壳找房** (https://sz.ke.com)
- 深圳小区列表页面
- 小区详情页面（基础信息、户型、均价）
- 成交记录页面（历史成交数据）

## 技术方案

### 技术栈
- **Node.js** - 与项目技术栈一致
- **Puppeteer** - 浏览器自动化，处理动态渲染和反爬
- **Cheerio** - HTML解析（备用方案）
- **node-cron** - 定时任务

### 爬取策略

#### 1. 小区列表爬取
```
入口: https://sz.ke.com/xiaoqu/
策略: 
- 按区域分页爬取（福田、南山、罗湖等10个区）
- 每页30个小区，共约3000+小区
- 使用Puppeteer模拟浏览器行为
```

#### 2. 小区详情爬取
```
入口: https://sz.ke.com/xiaoqu/{小区ID}/
数据项:
- 小区名称、地址、建成年份
- 均价、总户数、户型分布
- 地铁距离、配套设施
- 经纬度坐标
```

#### 3. 成交记录爬取（最近半年）
```
入口: https://sz.ke.com/chengjiao/
数据项:
- 成交房源、成交价格、成交日期
- 面积、户型、楼层
- 按小区聚合历史成交

时间范围:
- 仅爬取最近6个月的成交记录
- 计算半年成交均价用于AI推荐
- 减少数据量和爬取时间
```

### 反爬对策
1. **请求频率控制**: 随机延迟 2-5 秒
2. **User-Agent轮换**: 模拟真实浏览器
3. **IP代理池**: 必要时使用代理（付费服务）
4. **Cookie/Session管理**: 保持登录状态
5. **验证码处理**: 人工介入或打码平台

## 实现步骤

### Phase 1: 基础爬虫框架
1. 安装依赖: `puppeteer`, `cheerio`, `node-cron`
2. 创建 `scripts/crawler/` 目录
3. 实现基础爬虫类: `BaseCrawler`
4. 实现反爬策略: 延迟、UA轮换、代理

### Phase 2: 小区列表爬取
1. 实现 `DistrictCrawler` - 按区域爬取小区列表
2. 实现分页处理逻辑
3. 数据清洗和去重
4. 存储到 `communities.json`

### Phase 3: 小区详情爬取
1. 实现 `CommunityDetailCrawler` - 爬取单个小区详情
2. 并发控制（最多5个并发）
3. 错误重试机制（失败3次后跳过）
4. 补充缺失数据

### Phase 4: 成交记录爬取（最近半年）
1. 实现 `TransactionCrawler` - 爬取成交记录
2. **时间筛选**: 仅爬取最近6个月的成交数据
3. **数据聚合**: 按小区计算半年成交均价和成交量
4. 存储到 `transactions.json`
5. **增量更新**: 每次只爬取新增成交记录

### Phase 5: 数据合并与更新
1. 创建 `DataMerger` - 合并多源数据
2. 生成 `preloaded-data.js` 供小程序使用
3. 实现增量更新逻辑
4. 添加数据验证和清洗

### Phase 6: 定时任务与监控
1. 配置 `node-cron` 定时任务
2. 每周日凌晨2点执行全量更新
3. 每日凌晨3点执行增量更新（成交记录）
4. 添加日志记录和异常告警

## 使用方式

### 手动执行
```bash
# 完整爬取（首次执行）
node scripts/crawler/index.js --full

# 增量更新（仅更新成交记录）
node scripts/crawler/index.js --incremental

# 仅更新指定区域
node scripts/crawler/index.js --district 南山
```

### 定时任务
```bash
# 添加到系统定时任务（Linux/Mac）
crontab -e

# 每周日凌晨2点全量更新（小区列表+最近半年成交）
0 2 * * 0 cd /path/to/project && node scripts/crawler/index.js --full

# 每日凌晨3点增量更新（仅新增成交记录，最近半年）
0 3 * * * cd /path/to/project && node scripts/crawler/index.js --incremental
```

## 风险与应对

| 风险 | 可能性 | 应对方案 |
|-----|-------|---------|
| 贝壳找房反爬升级 | 高 | 使用代理池、降低频率、模拟真人行为 |
| 网站结构变更 | 中 | 监控页面结构、快速适配、数据校验 |
| 数据量过大 | 中 | 分片爬取、增量更新、数据压缩存储 |
| 法律风险 | 低 | 遵守robots.txt、控制频率、仅用于研究 |

## 验证方案

1. **数据完整性检查**: 确保每个小区都有基础信息
2. **价格合理性检查**: 均价在合理范围内（1-20万/㎡）
3. **坐标有效性检查**: 经纬度在深圳范围内
4. **人工抽样验证**: 随机抽取10%数据人工核对

4. AI推荐使用真实准确的候选数据（基于半年成交均价）
