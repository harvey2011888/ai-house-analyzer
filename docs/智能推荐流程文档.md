# 深圳购房助手 - 智能推荐流程文档

## 一、整体架构

```
用户交互层 (UI)
    │
    ├─ pages/chat/chat.js          # 需求收集
    ├─ pages/recommendation/recommendation.js  # 结果展示
    └─ pages/community-detail/community-detail.js  # 详情页

业务逻辑层 (Logic)
    │
    ├─ utils/ai-service.js         # AI推荐服务
    ├─ utils/local-recommend.js    # 本地推荐算法
    └─ utils/data-loader.js        # 数据加载器

数据层 (Data)
    │
    ├─ data/communities-data.js    # 200个小区数据
    ├─ data/communities.json.js    # 原始JSON数据
    └─ data/preloaded-data.js      # 预加载数据(20个热门)
```

## 二、智能推荐完整流程

### 阶段1: 需求收集 (chat.js)

**文件**: `pages/chat/chat.js`

用户通过8个步骤填写购房需求，每页显示2个问题：

| 步骤 | 问题ID | 类型 | 选项/范围 | 默认值 |
|------|--------|------|-----------|--------|
| 1 | budget | 滑块 | 300-1500万，步长50万 | 500万 |
| 2 | area | 滑块 | 40-200㎡，步长5㎡ | 90㎡ |
| 3 | propertyType | 多选 | 商品房/公寓/别墅/经济适用房/不限 | - |
| 4 | buildingAge | 多选 | 5年以内/5-10年/10-20年/20年以上 | - |
| 5 | bathroom | 多选 | 1卫/2卫/3卫及以上 | - |
| 6 | layout | 多选 | 1室/2室/3室/4室+ | - |
| 7 | district | 多选 | 福田/南山/罗湖/宝安/龙岗/龙华/盐田/坪山/光明/大鹏新区/深汕合作区 | - |
| 8 | priorities | 多选 | 学区/地铁/商业/医疗/环境/升值潜力 | - |

**用户答案格式示例**:
```javascript
const answers = {
  budget: 500,              // 预算500万
  area: 90,                 // 面积90㎡
  layout: ['3室', '4室+'],  // 户型偏好
  district: ['坪山'],       // 目标区域
  priorities: ['地铁', '学区'], // 核心诉求
  // ... 其他字段
};
```

**提交时保存到本地存储**:
```javascript
wx.setStorageSync('last_requirement', answers);
wx.navigateTo({ url: '/pages/recommendation/recommendation' });
```

#### 3.3 平均面积计算逻辑

根据**最大户型**计算平均面积（使用小区ID作为随机种子，保证结果一致）：

```javascript
const seed = parseInt(c.id) || 0;
const pseudoRandom = (min, max) => min + (seed % (max - min + 1));

if (layoutTypes.includes('4室') || layoutTypes.includes('5室')) {
  avgArea = 110 + pseudoRandom(0, 50); // 110-160㎡
} else if (layoutTypes.includes('3室')) {
  avgArea = 85 + pseudoRandom(0, 35);  // 85-120㎡
} else if (layoutTypes.includes('2室')) {
  avgArea = 65 + pseudoRandom(0, 25);  // 65-90㎡
} else if (layoutTypes.includes('1室')) {
  avgArea = 45 + pseudoRandom(0, 20);  // 45-65㎡
}
```

### 阶段4: 硬性条件筛选 (local-recommend.js)

**文件**: `utils/local-recommend.js` 中的 `filterByRequirements()` 函数

#### 4.1 筛选流程

对200个小区逐一进行4个条件的**硬性过滤**（AND关系，全部满足才能通过）：

```
小区列表 (200个)
    │
    ▼
[条件1: 区域匹配] ──► 排除不在目标区域的小区
    │
    ▼
[条件2: 户型匹配] ──► 排除不包含目标户型的小区
    │
    ▼
[条件3: 面积匹配] ──► 排除平均面积不在范围内的小区
    │
    ▼
[条件4: 预算匹配] ──► 排除总价不在预算内的小区
    │
    ▼
候选小区列表 (N个)
```

#### 4.2 各条件详细逻辑

**条件1: 预算筛选**
```javascript
const unitPrice = priceInfo.current_avg_price;           // 单价(元/㎡)
const totalPriceMin = (unitPrice * community.avg_area) / 10000; // 总价(万)

if (minBudget !== undefined && totalPriceMin < minBudget) return false;
if (maxBudget !== undefined && totalPriceMin > maxBudget) return false;
```

**条件2: 面积筛选**
```javascript
if (minArea !== undefined && community.avg_area < minArea) return false;
if (maxArea !== undefined && community.avg_area > maxArea) return false;
```

**条件3: 户型筛选**（支持"4室+"特殊匹配）
```javascript
if (layout === '4室+') {
  // "4室+" 匹配 "4室" 或 "5室"
  return community.layout_types.some(lt => lt.includes('4室') || lt.includes('5室'));
}
// 其他户型直接匹配
return community.layout_types.includes(layout);
```

**条件4: 区域筛选**（处理"新区"特殊名称）
```javascript
const userDistrict = d.replace('区', '').replace('新区', '');
return userDistrict === community.district;
```

#### 4.3 筛选结果示例

假设用户需求：
```javascript
{
  minBudget: 300, maxBudget: 600,
  minArea: 110, maxArea: 150,
  district: ['坪山'],
  layouts: ['3室', '4室+']
}
```

筛选过程：
```
初始: 200个小区
区域筛选后: 17个坪山小区
户型筛选后: 10个含3室/4室的小区
面积筛选后: 7个面积110-150㎡的小区
预算筛选后: 7个总价300-600万的小区
─────────────────────────────
最终候选: 7个小区
```

### 阶段5: 匹配度评分 (local-recommend.js)

**文件**: `utils/local-recommend.js` 中的 `calculateMatchScore()` 函数

对筛选通过的候选小区计算 **0-100分** 的匹配度分数。

#### 5.1 评分维度与权重

| 维度 | 权重 | 计算方式 |
|------|------|----------|
| **预算匹配度** | 30% | `1 - \|总价 - 预算中点\| / (预算范围/2)` |
| **面积匹配度** | 25% | `1 - \|面积 - 面积中点\| / (面积范围/2)` |
| **户型匹配度** | 20% | 匹配的户型数 / 用户选择的户型数 |
| **配套匹配度** | 25% | 地铁+学校+医院+商场的综合评分 |

#### 5.2 各维度详细计算

**预算匹配度 (30%)**:
```javascript
const totalPrice = (unitPrice * community.avg_area) / 10000;
const budgetMid = (minBudget + maxBudget) / 2;      // 预算中点
const budgetRange = maxBudget - minBudget;          // 预算范围

// 距离中点越近分数越高
let budgetScore = Math.max(0, 1 - Math.abs(totalPrice - budgetMid) / (budgetRange / 2));
score += budgetScore * 30;
```

**面积匹配度 (25%)**:
```javascript
const areaMid = (minArea + maxArea) / 2;
const areaRange = maxArea - minArea;

let areaScore = Math.max(0, 1 - Math.abs(community.avg_area - areaMid) / (areaRange / 2));
score += areaScore * 25;
```

**户型匹配度 (20%)**:
```javascript
const matchedLayouts = layouts.filter(layout => {
  if (layout === '4室+') {
    return community.layout_types.some(lt => lt.includes('4室') || lt.includes('5室'));
  }
  return community.layout_types.includes(layout);
});

let layoutScore = matchedLayouts.length / layouts.length;
score += layoutScore * 20;
```

**配套匹配度 (25%)**:
```javascript
let facilityPoints = 0;
let maxPoints = 4;

// 地铁 (1-1.5分)
if (facilities.metro && facilities.metro.name) {
  facilityPoints += 1;
  if (facilities.metro.distance < 500) facilityPoints += 0.5;
}

// 学校 (1-1.5分)
if (facilities.schools && facilities.schools.length > 0) {
  facilityPoints += 1;
  if (facilities.schools.length >= 2) facilityPoints += 0.5;
}

// 医院 (0.5分)
if (facilities.hospitals && facilities.hospitals.length > 0) facilityPoints += 0.5;

// 商场 (0.5-1分)
if (facilities.malls && facilities.malls.length > 0) {
  facilityPoints += 0.5;
  if (facilities.malls.length >= 2) facilityPoints += 0.5;
}

let facilityScore = Math.min(1, facilityPoints / maxPoints);
score += facilityScore * 25;
```

#### 5.3 评分示例

假设用户预算400-600万，面积80-100㎡：

| 小区 | 总价 | 面积 | 户型 | 配套 | 计算过程 | 得分 |
|------|------|------|------|------|----------|------|
| A | 450万 | 90㎡ | 3室 | 地铁+学校 | (0.5×30)+(1.0×25)+(1.0×20)+(0.75×25) | **81** |
| B | 550万 | 85㎡ | 3室 | 地铁 | (0.5×30)+(0.75×25)+(1.0×20)+(0.5×25) | **69** |
| C | 350万 | 110㎡ | 2室 | 学校+医院 | (0.25×30)+(0.5×25)+(0×20)+(0.5×25) | **33** |

### 阶段6: 生成推荐理由 (local-recommend.js)

**文件**: `utils/local-recommend.js` 中的 `generateReasons()` 函数

为每个匹配的小区生成 **最多3条** 推荐理由。

#### 6.1 推荐理由类型

**预算理由**:
```javascript
if (totalPrice <= maxBudget && totalPrice >= minBudget) {
  if (totalPrice <= budgetMid) {
    reasons.push(`总价约${totalPrice}万元，在您的预算范围内性价比较高`);
  } else {
    reasons.push(`总价约${totalPrice}万元，符合您的预算预期`);
  }
}
```

**面积理由**:
```javascript
if (community.avg_area >= minArea && community.avg_area <= maxArea) {
  reasons.push(`平均面积${community.avg_area}㎡，符合您的面积需求`);
}
```

**户型理由**:
```javascript
reasons.push(`提供${matchedLayouts.join('、')}户型，满足您的居住需求`);
```

**配套理由**:
```javascript
// 地铁
if (metro.distance < 500) {
  reasons.push(`距${metro.name}仅${metro.distance}米，交通便利`);
}

// 学校
reasons.push(`周边教育资源丰富，有${schoolNames}等`);

// 商场
reasons.push(`生活配套完善，临近${mallNames}`);
```

**价格趋势理由**:
```javascript
if (price.yoy_change > 0) {
  reasons.push(`房价同比上涨${price.yoy_change}%，保值增值潜力较好`);
} else if (price.yoy_change < 0) {
  reasons.push(`房价同比下降${Math.abs(price.yoy_change)}%，入手时机较好`);
}
```

**楼龄理由**:
```javascript
const buildingAge = currentYear - community.build_year;
if (buildingAge <= 5) {
  reasons.push('次新小区，房屋状况较新');
} else if (buildingAge <= 15) {
  reasons.push('小区成熟，配套设施完善');
}
```

### 阶段7: AI智能推荐 (ai-service.js)

**文件**: `utils/ai-service.js`

#### 7.1 AI推荐流程

```
本地筛选结果 (N个候选小区)
    │
    ▼
[限制数量] ──► 最多30个候选（避免API过载）
    │
    ▼
[构建Prompt] ──► 系统提示词 + 用户需求 + 候选小区数据
    │
    ▼
[调用通义千问API] ──► qwen-turbo模型
    │
    ▼
[解析AI响应] ──► JSON格式推荐结果
    │
    ▼
[合并数据] ──► AI排序 + 本地数据补充
    │
    ▼
展示TOP3推荐
```

#### 7.2 Prompt构建

**系统提示词**:
```
你是一位拥有10年经验的深圳购房顾问专家，精通深圳各区域的房价、配套、学区、交通等情况。

分析维度：
1. 预算匹配度 - 小区均价是否在用户预算范围内
2. 面积匹配度 - 小区平均面积是否符合用户需求
3. 户型匹配度 - 小区是否提供用户需要的户型
4. 区域匹配度 - 小区是否在用户期望的区域
5. 配套完善度 - 学区、地铁、商业、医疗等配套

输出要求（严格JSON格式）：
```

### 阶段8: 结果展示 (recommendation.js)

**文件**: `pages/recommendation/recommendation.js`

#### 8.1 数据合并

将AI推荐结果与本地小区数据合并：

```javascript
function enrichRecommendations(aiRecommendations, candidates) {
  return aiRecommendations.map((rec, index) => {
    const communityData = candidates.find(c => c.id === rec.community_id);
    const priceInfo = prices.find(p => p.community_id === rec.community_id);
    const facilityInfo = facilities.find(f => f.community_id === rec.community_id);

    return {
      id: rec.community_id,
      rank: index + 1,
      name: communityData.name,
      district: communityData.district,
      matchScore: rec.match_score,
      reasons: rec.reasons,
      pros: rec.pros,
      cons: rec.cons,
      currentAvgPrice: priceInfo.current_avg_price,
      avgArea: communityData.avg_area,
      layoutTypes: communityData.layout_types,
      buildYear: communityData.build_year,
      metroDistance: facilityInfo.metro.distance
    };
  });
}
```

## 四、关键算法说明

### 4.1 总价计算公式

```
总价(万元) = 单价(元/㎡) × 平均面积(㎡) / 10000
```

示例：
- 单价: 44,414元/㎡
- 平均面积: 149㎡
- 总价: 44,414 × 149 / 10000 = 661.8万

### 4.2 匹配度评分公式

**预算匹配度**:
```
budgetScore = max(0, 1 - |totalPrice - budgetMid| / (budgetRange / 2))
```

**面积匹配度**:
```
areaScore = max(0, 1 - |avg_area - areaMid| / (areaRange / 2))
```

**户型匹配度**:
```
layoutScore = matchedLayouts / totalLayouts
```

**配套匹配度**:
```
facilityScore = min(1, facilityPoints / 4)
```

**总分**:
```
totalScore = budgetScore × 30 + areaScore × 25 + layoutScore × 20 + facilityScore × 25
```

### 4.3 数据一致性保证

- **面积计算**: 使用小区ID作为随机种子，保证同一小区每次生成的面积一致
- **价格历史**: 基于当前价格生成±1%波动的历史数据
- **配套设施**: 根据区域名称自动生成，保证一致性

## 五、异常处理

### 5.1 无匹配结果

当筛选后候选小区为0时：
```javascript
if (filteredCommunities.length === 0) {
  return {
    recommendations: [],
    advice: '抱歉，根据您的需求条件，暂未找到匹配的小区。建议您适当放宽预算范围或面积要求，以便获得更多选择。'
  };
}
```

### 5.2 AI调用失败

自动降级到本地推荐算法，不中断用户体验。

### 5.3 数据加载失败

- 优先加载200个小区数据
- 失败时降级到20个预加载小区
- 再失败时使用空数据
